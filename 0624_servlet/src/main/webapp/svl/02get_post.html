form : 서버로 보내고자 하는 데이터들을 묶는 태크

action : 서버의 주소:포트/프로젝트/프로그램id 등을 넣는 부분 : 데이터를 전송할 주소

method : 전송 방법 : get과 post가 대표적

get : 주소창에 서버로 전송할 data를 첨부하는 방식
get의 단점 : 주소창에 data가 보임 -> 보안 취약(정보유출), 256byte만 신뢰 가능.
get의 장점 : 속도가 post보다 빠르다.

QueryString : get 방식에서 서버로 보내는 주소창에 보이는 데이터. QueryString 만드는 방법 : 시작은 ?, dataName=dataValue, 두 개 이상 연결은 & QueryString의 예 : ?mid=aaaaaaa&mpwd=111111111111&tel=010&email=daum

post : 주소창에 서버로 전송할 data가 보이지 않도록 하는 전송 방식.
post의 단점 : 속도가 get보다 느리다.
post의 장점 : 주소창에 data가 보이지 않는다 -> 보안이 get보다 좋다, 대용량 data 전송 가능.
